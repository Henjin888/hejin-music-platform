# XMUSIC 歌手融资平台 - 开发规格

## 核心定位
**音乐现金流的资产化和分配层**
- 不跟 Spotify 抢流量、不烧存储带宽
- 只做：音乐现金流的资产化和分配层
- 成本结构轻，模式适合全球任何平台上的歌手
- 不受单一流媒体平台的限制

## 核心概念

### 每个歌手 = 一个小项目
每个项目都有四个角色：
1. **创始人（歌手）** - 项目发起人，决定份额分配
2. **运营合伙人（经纪人）** - 负责运营，拿 Carry + XMUSIC 激励
3. **LP（投资人）** - 有真收益权 + 治理权
4. **社区（粉丝）** - 从普通支持者变成"共创人"，可升级成更核心角色

### XMUSIC 的作用
把他们的权利 & 收益分配都固化在链上

## 功能需求

### 1. 登录系统
支持多种登录方式：
- Web3 钱包（MetaMask、WalletConnect、Rainbow 等）
- 苹果 ID 账号
- 谷歌 ID 账号
- 手机号码
- 其他主流社交媒体账号（Twitter、Discord 等）
- 一键登录注册，弹出绑定手机号码

### 2. 首页设计
完全参考 Spotify 排版：
- 深色主题（#121212 背景）
- 绿色品牌色（#1DB954）
- 左侧导航栏
- 顶部搜索栏
- 卡片式内容展示
- 底部播放控制栏（可选）

### 3. 地图功能
- 地图寻找附近的音乐人
- 帮助附近的投资人快速找到附近的歌手
- 基于地理位置的项目发现

### 4. 取消的功能
- ❌ 直播功能（不需要）

### 5. 多语言支持
- 中文（默认）
- 英文
- 其他语言可扩展

## 技术栈
- Next.js 14 + React + TypeScript
- Tailwind CSS
- Web3 集成（RainbowKit + wagmi）
- 地图 API（Mapbox 或 Google Maps）
- 多语言（next-i18next）

## 页面结构

```
/                    # 首页（Spotify 风格）
/login               # 登录页（多种登录方式）
/explore             # 探索（地图 + 列表）
/project/[id]        # 项目详情
/artist/[id]         # 艺人详情
/dashboard           # 投资面板
/create              # 创建项目
/map                 # 地图找音乐人
/settings            # 设置（语言切换等）
```
