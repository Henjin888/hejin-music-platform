# XMUSIC 融资功能开发需求

## 背景
聚焦 “/focus 融资” 方案，XMUSIC 需要支撑平台代币 $XMU 的预售、治理与透明化运作，同时兼顾创作者参与和风险披露。本需求整理 A1/A2/A3/A5 的交付物，形成可落地的产品与技术开发清单。

## 关键交付物
- $XMU 代币与金库治理体系
- 预售与 NFT Pass 支持能力
- 公开透明的合规披露与看板
- 部署至建池的自动化/半自动化运维流程
- 7 天内可执行的内容发布与行动工具

## 产品与功能需求
### 1. 代币与金库模块
- 在 Solana (SPL) 上部署 $XMU，初始总量 1,000,000,000，9 位小数。
- 支持一次性转移铸造/冻结/升级权限至 3/5 多签。
- 代币分配表管理：
  - 社区与增长 35%，36 个月线性释放。
  - 生态补贴 25%，按提案拨付。
  - 金库储备 15%，锁定 3 个月后按需拨付。
  - LP 做市 10%，按照公开规则分期注入。
  - 早期贡献者 10%，3 个月悬崖后 24–36 个月线性释放。
  - 顾问合作 5%，12–24 个月线性释放。
- 金库后台需支持提案创建、预算审批与 Tx 记录，默认月度拨付上限不超预售净额的 30%。

### 2. 预售与 NFT Pass
- 预售额度不超过总量的 2%，拆分为早鸟/公众两档（各 1%），定价与人数上限可配置。
- 解锁规则：悬崖 1 个月，随后 5 个月线性释放；需在前端清晰展示。
- 所有募集资金需托管于多签金库，实时公开地址与进度。
- 配套 NFT Pass 三档（Creator/Pro/Guild），展示权益（活动门票、白名单优先、治理门票等），明确无分红承诺。
- 预售 FAQ 与规则页面需同步上线，支持更新公告、倒计时与每日进度推送。

### 3. 治理与透明化
- 公布合约地址、多签地址、权限回收 Tx 的公告模块。
- 金库看板：字段含日期、用途、金额、提案链接、签名 Tx、负责人，支持周报导出。
- 支持提案讨论（≥72 小时）、投票与执行状态记录。
- 事件应急流程工具：异常上报、暂停拨付、快照、公告模版。

### 4. 合规与话术资产
- 官网/白皮书模板内置风险话术：
  - “XMU 用于平台治理、访问与激励，不代表股权、债权或任何形式的收益权。”
  - “不承诺任何回报，价格波动与持有风险由持有者自行承担。”
  - “任何代币或 NFT 的购买均为对项目使用权与社区建设的支持，而非投资合同。”
- 红线词替换库嵌入内容 CMS：投资→平台权益、收益→治理参与、保本→存在价格波动等。
- 披露清单页面：代币分配、解锁时间表、金库预算规则、重大变更流程、应急预案。
- 反女巫规则配置：轻 KYC/社媒绑定、地址筛查、黑名单与申诉入口。

### 5. 运维 SOP 与自动化
- 部署核对表工具：预部署、部署、发布、运维、回滚五阶段 checklist。
- 发布公告模板（合约 & 权限回收、预售 & LP）可在后台一键生成。
- 部署演练记录（测试网/主网）、截图或录屏上传存档。
- 每周自动生成金库周报草稿，推送至指定频道/邮箱。

### 6. 行动内容支持
- 生成式内容或模板中心，包含：一页募资文案、AMA 大纲、KOL 外联私信模板、预售倒计时素材占位。
- 任务排程模块，对应 7 天行动单，支持标记完成与负责人。
- 等候名单表单与数据导出功能，对接邮件/社区工具。

## 技术要求
- Solana 钱包与多签集成（如 Squads/Realms），确保权限迁移可验证。
- 前端需支持链上数据可视化（解锁曲线、金库余额、Tx feed）。
- 后端需具备事件监听（权限变更、拨付、LP 操作）与提醒机制。
- 数据存储遵循隐私要求，KYC 数据分级加密处理。
- 所有模块需提供 API/SDK 接口，与现有 XMUSIC 平台无缝对接。

## 验收标准
- 多签权限转移与金库拨付流程全链路演示通过。
- 预售页面上线并通过白名单、托管、释放规则测试。
- 金库看板与周报自动化数据准确，公开字段齐全。
- 合规披露页面无敏感词，风险话术默认启用。
- 行动工具支撑 7 天内内容发布与任务跟踪，文案模板可直接使用。
